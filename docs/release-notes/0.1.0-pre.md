# Maku 0.1.0 pre-release handoff

Release-readiness implementation baseline: `2f6f22d` and its ancestors. The
immutable source revision published in browser `release.json` is supplied by
the eventual release commit/tag through `MAKU_SOURCE_REVISION`; development
snapshots intentionally report `development`.

## Coherent version set

- Rust: `maku`, `maku-render-touhou`, `maku-player`, `maku-web` 0.1.0
- npm/browser: `@mlegls/maku` 0.1.0
- render frame ABI: 1
- Rust/MSRV: 1.97
- wasm-pack: 0.15.0
- Bun: 1.3.14

The wasm bytes, wasm-bindgen JavaScript/declarations, wrapper, and
`release.json` are one artifact. Consumers must not copy only the wasm file.

## Migration notes

- The Rust workspace moved from `proto/` to `crates/`.
- Embedded libraries moved from `cards/lib/` to the canonical
  `crates/core/lib/` package source.
- The unpublished `maku-mesh-touhou` package became
  `maku-render-touhou`; Rust imports use `maku_render_touhou`.
- Public embedding code should use `maku::host`, `maku::source`, and
  `maku::render`. Interpreter, simulation storage, session internals, and old
  module paths are explicitly unstable.
<!-- compatibility-migration -->
- `value-or` was removed; use `default`.
<!-- compatibility-migration -->
- Pre-release `spawn-bullet`, `spawn-shot`, `spawn-enemy`, `spawn-boss`, and
  `spawn-player` aliases were removed; use `bullet`, `shot`, `enemy`, `boss`, and `player`.
- Direct render-row `:facing`, `:opacity`, and `:pts` aliases were removed;
  use `:theta`, `:alpha`, and `:points`. Touhou entity metadata named
  `:facing` and `:opacity` remains valid and is translated by the library.
- The unscoped npm name `maku` is unrelated; browser releases use
  `@mlegls/maku`.

## Docs/demo handoff

The docs/demo change should build from the release commit, set
`MAKU_SOURCE_REVISION` to that commit, retain the generated `release.json`, and
synchronize wasm, glue, wrapper, renderer, cards, and documentation as a
single manifest. See `docs/releasing.md` and `scripts/check-generated.sh`.
