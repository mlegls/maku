# Maku 0.2.0

Maku 0.2 consolidates the Rust distribution around one SDK while preserving
language semantics, deterministic replay, ordered typed render transport, and
Touhou frame ABI 1.

## Distribution

- Rust SDK: `maku` 0.2.0
- Browser: `@mlegls/maku` 0.2.0, with bundled `touhou` pack contract 1
- Native player: platform binaries attached to GitHub Release `v0.2.0`
- Internal only: `maku-player`, `maku-web`, and benchmark producer packages

The Cargo default feature set has no genre renderer, Macroquad, or wasm-bindgen
dependency. Enable `touhou` for the standard render pack or `macroquad` for the
Touhou pack plus native compatibility adapter.

## Rust migration from 0.1

| 0.1 dependency/import | 0.2 replacement |
|---|---|
| `maku = "0.1"` | `maku = "0.2"` |
| `maku-render-touhou = "0.1"` | `maku = { version = "0.2", features = ["touhou"] }` |
| `maku_render_touhou::*` | `maku::touhou::*` |
| `maku_player::macroquad_compat::*` | `maku::macroquad::*` with feature `macroquad` |
| `maku-web` Cargo dependency | install `@mlegls/maku` for browser use |
| `cargo install maku-player` | download a player binary from GitHub Releases |

The auxiliary Cargo packages stop at 0.1.0 and are yanked only after verified
0.2 replacements are available. Their source and registry records remain
historical release evidence.

## Render-pack inclusion

The official npm package and player bundle Touhou. This is a declared curated
pack set, not a promise that every frontend contains every future pack. Future
frontends expose their bundled pack IDs/contract versions and reject cards that
require an absent pack.
